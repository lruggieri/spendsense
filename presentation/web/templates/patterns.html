About Patterns

Patterns automatically categorize transactions by matching keywords in descriptions. You can create rules using simple operators without needing to know regex syntax.

Available Operators:

  • Starts with - Transaction must begin with this keyword
  • Ends with - Transaction must end with this keyword
  • Not starts with - Transaction must NOT begin with this keyword
  • Contains (OR) - Match if transaction contains ANY of these keywords
  • Must contain (AND) - Transaction must contain ALL of these keywords
  • Must not contain (NOT) - Transaction must NOT contain this keyword

Pattern Priority: First matching pattern wins. Drag patterns to reorder priority.
